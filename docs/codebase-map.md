## Workspace Shape

- `shared/`
  - Runtime schemas and TypeScript contracts shared by server and web.
  - Package runtime entrypoint resolves from `shared/dist`, while schema/type source of truth stays in `shared/src`.
  - Start here when route payload, response shape, enum value, or validation source of truth is question.
- `server/`
  - Fastify API, SQLite access, Drizzle schema/migrations, deterministic Today aggregation, GCal inbound sync, LLM gateway.
  - Optional Telegram long-poll worker handles real needs-review push replies.
  - Start here when behavior touches persistence, sync, route wiring, or integration boundaries.
- `web/`
  - React + Vite PWA shell. Current user-facing surface centers on `/today`.
  - Local dev default: Vite on `5173`, backend proxy target on `3100`.
  - Start here when behavior is UI state, fetch flow, or PWA output.
- `docs/`
  - Product spec and design docs.
- `.review/`
  - Cycle plans, advisor feedback, review verdicts, status files.
- `.claude/`, `.codex/`, `.agents/`
  - Agent workflow rules, hooks, local skills.
- `multi-agent-starter/`
  - Reference/template material. Not main Cairn runtime path.

Route layer:

- [server/src/routes/events.ts](/home/pi/cairn/server/src/routes/events.ts)
  - `POST /api/events` — event creation with optional personIds (transactional) and optional `mode` (in_person|remote|async; cycle-44). Invalid mode → 400 VALIDATION_ERROR (writes nothing).
  - `GET /api/events/:id` — event detail: event + people + annotations (newest-first) + compact thread + `scheduleBrief` (cycle-44 FR-BRF). Brief loads nearest prior same-thread event (only when this event has threadId+start) via `findNearestPriorThreadEvent`, its newest annotation via `pickNewestAnnotation`, and `preparations` via `findPreparationLinkData` → `buildPreparations` (cycle-45 FR-BRF-04: resources linked to the event/its thread/the prior event), and `preparationSuggestions` via `buildPreparationSuggestions(event, thread, preparations)` (cycle-47 FR-BRF-04: deterministic keyword suggestions, suppressing already-prepared items), then assembles with `buildScheduleBrief`. Read-only.
  - `PATCH /api/events/:id/status` — status update (planned/confirmed/done/cancelled/moved/late). Deterministic; no LLM.
  - `PATCH /api/events/:id/thread-node` — thread node inline edit (cycle-50 FR-THR-06). Body `PatchThreadEventNodeRequest` (strict partial: title/type/location/mode only; ≥1 field). id→400, body→400, unknown→404, GCal-imported (`source='gcal'`)→409 `EXTERNAL_EVENT_READ_ONLY` before any write. Mutates only allowed columns via `updateEventThreadNode`; start/end/status/threadId/source untouched. Returns `{ event }`.
  - `PATCH /api/events/:id/schedule-prompt/dismiss` — hide an unscheduled-event schedule prompt for one Today date (cycle-61 FR-SLOT-06B/FR-TODAY-05). Body `DismissSchedulePromptRequest` (strict `{ dismissedOn: YYYY-MM-DD }`, real-date refine). id→400, body→400, unknown→404; `findEventById` then guarded `dismissSchedulePromptForDate` — if no eligible row matched (scheduled/external/cancelled/non-self-imposed/started)→409 `SCHEDULE_PROMPT_NOT_ELIGIBLE` with no write. Idempotent. Writes only `schedule_prompt_dismissed_on`+`updated_at`. Returns `{ eventId, dismissedOn }`.
  - `PATCH /api/tasks/:id/thread-node` — thread node inline edit (cycle-50 FR-THR-06). Body `PatchThreadTaskNodeRequest` (strict partial: title/estMinutes/due/context/optional; ≥1 field; `due` a real calendar date or null). id→400, body→400, unknown→404. Mutates only allowed columns via `updateTaskThreadNode`; status/threadId untouched. Returns `{ task }`.
  - `PATCH /api/threads/:id/node-links/:linkId/confirm` — explicit firmness promotion (cycle-50 FR-THR-05). No body. `confirmThreadNodeLink` promotes a same-thread event/task `links` row to firmness='hard'+source='authored' together (never hard+inferred); idempotent (`reused=true`, no write) when already hard/authored. 404 when the link is unknown, cross-thread, or has a missing endpoint. Returns `{ link: ThreadNodeLink, reused }`.
  - `POST /api/events/:id/preparations` — manual one-line preparation (cycle-46 FR-BRF-04). Body `CreateEventPreparationRequest` `{ name }` (trimmed 1..120). Validates id/body → 400; `eventExists` → 404; calls `addEventPreparation` (one transaction: find-or-create `item` resource by exact (name,kind='item'), idempotently link to the event with firmness='hard'/reason='직접 추가', existing link NOT rewritten). Returns `{ resource, link, reusedResource, reusedLink }`; `201` new link, `200` exact link already existed; `400 DB_ERROR` on constraint failure. The next GET surfaces it as scope `event_direct`.
- [server/src/routes/tasks.ts](/home/pi/cairn/server/src/routes/tasks.ts)
  - Task creation and status patch APIs.
  - `GET /api/tasks/:id/slot-candidates?date&now&days` — READ-ONLY due-task slot preview (cycle-62 FR-SLOT-06C). Reuses `SlotCandidatesQuerySchema`; id→400, query→400, unknown→404; `isTaskPromptEligible(task, q.date)` fails→409 `TASK_SCHEDULE_PROMPT_NOT_ELIGIBLE`. Calls `generateTaskSlotCandidates(db, {threadId, estMinutes}, ...)` (duration = task est_minutes, NO 60-min fallback). No DB write, no event created. Returns `{ task, candidates }`.
  - `PATCH /api/tasks/:id/schedule-prompt/dismiss` — hide a due-task schedule prompt for one Today date (cycle-62). Body `DismissTaskSchedulePromptRequest` (strict `{ dismissedOn: YYYY-MM-DD }`, real-date refine). id→400, body→400, unknown→404; `dismissTaskSchedulePromptForDate` re-checks eligibility against the dismiss date→409 if ineligible. Idempotent. Writes ONLY `schedule_prompt_dismissed_on` (tasks have no `updated_at`). Returns `{ taskId, dismissedOn }`.
  - `POST /api/tasks/:id/schedule-block` — apply a due-task slot candidate (cycle-63 FR-SLOT-07A): create ONE scheduled Cairn block event and record it on the task. Body `ScheduleTaskBlockRequest` (.strict `{ date, now, days, start, end }`, end>start). id→400, body→400, unknown→404; `isTaskPromptEligible(task, body.date)` false→409 `TASK_SCHEDULE_PROMPT_NOT_ELIGIBLE`; `hasActiveScheduledBlock` true→409 `TASK_ALREADY_SCHEDULED`; recomputes `generateTaskSlotCandidates(body.now, body.date, body.days)` and requires an exact start+end match else→409 `TASK_SLOT_STALE`. On success the transactional `scheduleTaskBlock` inserts the event (`type='task'`, `mode='async'`, `source='cairn'`, `self_imposed=1`, `status='planned'`, title/thread from task) and updates ONLY `tasks.scheduled_event_id` — task status/due/estimate/optional/thread untouched, no `links` row. 201 `{ task, event }`. All failure paths write nothing. `hasActiveScheduledBlock` = marker→event live (start/end non-null, cairn, self_imposed=1, status∈planned|confirmed|done); cancelled/moved/missing resurface the prompt. `findDueTaskSchedulePrompts` also excludes active-block tasks.

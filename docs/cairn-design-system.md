# CAIRN — 디자인 시스템 명세

`cairn-today-production`에서 확립된 패턴을 나머지 화면(맥락·거울·여백·결정·feasibility)의 품질 기준선으로 고정한다. 모든 화면은 이 문서의 토큰·상태·애니메이션·컴포넌트·반응형 규칙을 따른다.

---

## 0. 디자인 원칙 (UX, 이 시스템 고유)

일반 앱과 매력의 원천이 정반대다. 목표는 **체류가 아니라 이탈** — 열어서, 보고, 닫는 30초 세션이 성공.

1. **이탈을 설계한다.** 모든 화면은 "처리하고 닫기"가 목표. 더 볼 것을 권하지 않음. 처리할수록 화면이 비어가는 게 보여야 함.
2. **침묵이 기본이자 보상.** 빈 화면("조용해")은 실패가 아니라 잘 돌아간다는 신호. 배지·빨간 점 남발 금지.
3. **모든 제안에 이유를 붙인다.** "팀 회의 이동 추천" + "← SangHyung 당일 OK". 이유 없는 제안은 신뢰를 못 쌓음.
4. **firmness가 시각적으로 즉시 읽힌다.** 실선/점선/흐림은 *얼마나 믿어도 되는가*. 추론을 확정처럼 보이게 금지.
5. **거울은 비추되 판단하지 않는다.** 기술(description)만, 질책 톤·경고색 금지.
6. **입력 부담은 탭 한 번 이하.** 폼을 만나는 순간 시스템이 죽음. 고르는/끄덕이는 컴포넌트만.

## 1. 시맨틱 토큰

모든 색은 hex가 아니라 *이름*을 참조한다. 라이트/다크는 같은 이름에 다른 값을 매핑(단순 반전 X, 재튜닝 O). A/B 온도도 같은 메커니즘.

### 1.1 토큰 셋

| 토큰 | 역할 |
| --- | --- |
| `--bg` | 최하단 배경 |
| `--surface` / `--raised` | 카드 / 한 단계 위 표면 |
| `--border` | 경계선 |
| `--text` / `--muted` / `--faint` | 본문 / 보조 / 흐림 |
| `--accent` / `--on-accent` | Cairn 시그널(now·강조) / 그 위 텍스트 |
| `--done` / `--moved` / `--cancelled` / `--late` | outcome 색 |
| `--firm-hard` / `--firm-soft` / `--firm-tent` | firmness(확정/추론/미정) |
| `--skel` | 스켈레톤 |
| `--shadow` | 시트·부유 요소 그림자 |

### 1.2 모드별 값

**다크 (기준):**
```
--bg:#0E1418  --surface:#161F24  --raised:#1C272D  --border:#26343B
--text:#E8E4D8  --muted:#7C8A8F  --faint:#566268
--accent:#E0B250  --on-accent:#0E1418
--done:#5BA88E  --moved:#D6A24E  --cancelled:#C05E52  --late:#9B7FB5
--firm-hard:#5BA88E  --firm-soft:#D6A24E  --firm-tent:#3A474D
```

**라이트 (재튜닝):**
```
--bg:#F5F2EA  --surface:#FFFFFF  --raised:#FBF8F0  --border:#DED8C9
--text:#1E2A2E  --muted:#5C6A6F  --faint:#9AA2A3
--accent:#B07A1E  --on-accent:#FFFFFF
--done:#2E8B6E  --moved:#B0791F  --cancelled:#AD453B  --late:#735B8F
--firm-hard:#2E8B6E  --firm-soft:#B0791F  --firm-tent:#C7C0AE
```

전환: 수동 토글 + OS `prefers-color-scheme` 따라가기(`system`). 그 이상 똑똑한 자동 전환은 안 함(혼란 방지).

⚠ outcome·firmness 색은 모드별 *따로 튜닝*. 단순 invert는 라이트에서 탁해짐 — 실기기 대비 검증 필수(특히 `--muted`, 칩 경계).

### 1.3 타이포

| 역할 | 폰트 | 쓰임 |
| --- | --- | --- |
| UI/display | Space Grotesk | 기본 — 모든 화면 |
| 데이터 | IBM Plex Mono | 시각·수치·날짜·ID |
| 회고(B) | Newsreader (serif) | **거울·조용 상태·완결 정산의 헤딩에만** 엄격 제한 |

세리프는 A/B 신호용이라 B 컨텍스트 밖으로 새지 않게.

## 2. A/B 온도 — 화면 목적이 자동 선택

사용자 토글 아님. 화면의 *목적*이 결정한다.

| | A — 실행·결정 | B — 회고·자기인식 |
| --- | --- | --- |
| 화면 | 오늘·feasibility·결정·여백 | 거울·완결 정산·과거 노드 주석 |
| 매력 | 조용하고 믿음직한 도구 | 내 생각을 적는 사적 공간 |
| 폰트 | Space Grotesk | 헤딩 Newsreader |
| 표면 | 기본 | 따뜻하게 틴트(`--surface` 재정의) |
| 여백·행간 | 고밀도 | 넉넉 |
| 체류 | 짧게(처리 후 떠남) | 머물러 돌아봄 |

구현: A를 뼈대로, B는 `.warm` 클래스가 `--surface`/`--raised`만 재정의 + 세리프 + 여백 확대. **레이아웃은 안 바꿈** — 온도(폰트·표면·여백)로만 표현해 한 화면에서 시끄럽게 안 부딪히게. thread 뷰에선 NOW선 위(A) / 아래(B)가 미묘한 온도차를 가질 수 있음.

```
[data-theme="dark"] .warm{ --surface:#1A211F; --raised:#212824; }
[data-theme="light"] .warm{ --surface:#FCFAF2; --raised:#F6F1E4; --bg:#F3EEE0; }
```

## 3. 화면 상태 — 모든 화면이 4종을 갖춘다

`cairn-today`에서 확립. 모든 데이터 화면은 아래를 구현한다.

| 상태 | 처리 | 톤·카피 원칙 |
| --- | --- | --- |
| **loading** | 스켈레톤(스피너 X) — 실제 카드 모양 회색 블록 pulse, stagger 120ms | 차분한 미학 유지 |
| **quiet (빈)** | **보상으로 디자인.** B 온도(세리프·여백), 중앙 정렬, 작은 done 원 | "오늘은 조용해 / 새로 생기면 올려둘게 / 닫고 네 일 해도 돼". 슬픈 일러스트 금지 |
| **live (채워짐)** | 우선순위 stack, 진입 stagger 애니메이션 | 처리할수록 비어감 |
| **error** | 인터페이스 목소리 + 복구 액션 | "X를 불러오지 못했어 / 원인·해법 / 마지막 정보는 그대로". 사과·모호 금지. [다시 시도] |

각 화면의 quiet/error 카피는 그 화면 맥락으로 바꾸되 톤 원칙은 동일.

## 4. 애니메이션

은은하게, 의미 있는 곳에만. 과하면 AI생성 느낌.

| 모션 | 용도 | 스펙 |
| --- | --- | --- |
| `rise` | 카드 진입 | opacity 0→1 + translateY 8→0, .3s ease, stagger 55ms |
| `up` | 바텀시트 등장 | translateY 100%→0, .26s cubic-bezier(.2,.8,.2,1) |
| `fade` | 백드롭 | opacity 0→1, .2s |
| `pulse` | 스켈레톤 | opacity .5↔1, 1.3s 무한 |
| `pop` | 체크 완료 | scale .6→1.15→1, .25s |
| press | 버튼 누름 | `:active` scale .97, .08s |
| width | 게이지·바 | width .4s ease |

`prefers-reduced-motion` 존중(모션 축소 시 진입·pop 제거, 상태 변화는 유지).

## 5. 컴포넌트 — "타이핑 < 탭 < 무행동"

push 시스템이라 대부분 입력이 *반응*. 폼처럼 채우는 컴포넌트 피하고 고르는/끄덕이는 것으로.

| 컴포넌트 | 언제 | 스펙 |
| --- | --- | --- |
| **세그먼트** | 단일 선택 2–4 (계획/임박, 테마) | 한 번에 다 보임, 선택=`--accent` 배경. 드롭다운 아님 |
| **드롭다운/콤보** | 단일 선택 5+ | 화면 절약용. 그제야 |
| **칩 토글** | 다중 선택 (reason_tags) | radius 20, 선택=accent 채움. 체크박스 리스트보다 빠름 |
| **토글 스위치** | on/off (watcher armed) | 즉시 반영, 50×30, thumb 24 |
| **슬라이더 + 실시간 숫자** | 연속값 (feasibility 파라미터) | thumb 22(터치), 옆에 mono 숫자, 만지면 결과 즉시 바뀜 |
| **확인 버튼** | firmness 승격(soft→hard) | 단일 액션 "확인 ✓", 끄덕임 한 번 |
| **카드** | 정보 단위 | `--surface`, radius 14, 좌측 accentBar로 긴급도 |
| **BigBtn** | 화면당 주 액션 | 하단 고정, full-width, `--accent`, minHeight 48 |
| **바텀시트** | 상세·선택 (결정·인물·주석) | 아래서 up, 드래그 핸들, 백드롭 탭 닫기 |
| **자유 입력** | push 답·자연어 생성 **에만** | 그 외 화면엔 없음 |

전 컴포넌트 **터치 타겟 ≥ 44px**.

## 6. 모바일 우선 (사용 70% 모바일)

반응형이 아니라 **mobile-first, PC는 확장**.

### 6.1 레이아웃

- **하단 탭바**(오늘/맥락/거울/여백) — 엄지 존. 활성=`--accent`.
- **단일 컬럼 stack** — 데모의 좌/우 패널은 모바일에서 위아래로 쌓거나 바텀시트로.
- **PC** — 단일 컬럼을 넓히고 옆에 패널을 *덧붙임*. 모바일 레이아웃 안 부숨.

### 6.2 인터랙션

- 주요 액션은 **화면 하단**(상단은 엄지 안 닿음).
- **바텀시트 = 주력 패턴**: 결정 인터럽트, 인물 상세, 노드 주석 펼침 전부 아래서 올라옴.
- **스와이프**: push 카드 옆으로 밀어 snooze, 2분 태스크 밀어 완료(탭보다 빠름). *프로토타입엔 버튼 어포던스로 대체, 실제 앱은 터치 드래그 구현 필요.*
- 한 화면에 다 보이게, 핵심 액션 엄지 한 번. 모바일 70%라 30초·이탈 설계가 더 강함.

## 7. 화면별 적용 체크리스트

나머지 화면 구현 시 아래를 만족해야 "기준선 통과".

- [ ] 4상태(loading/quiet/live/error) 모두 구현, 카피는 화면 맥락 + 톤 원칙
- [ ] 모든 색이 시맨틱 토큰 참조 (하드코딩 hex 0)
- [ ] 라이트/다크 양쪽 검증 (특히 muted·경계 대비)
- [ ] A/B 온도 올바름 (실행=A / 회고=B), 세리프는 B 헤딩에만
- [ ] firmness 시각(실선/점선/흐림) 일관
- [ ] 컴포넌트는 "고르는/끄덕이는" 것, 자유 입력 최소, 터치 44px+
- [ ] 주 액션 하단, 상세는 바텀시트, 진입 stagger
- [ ] `prefers-reduced-motion` 존중
- [ ] 빈 상태가 *보상*으로 느껴지는가 (실패 아님)
- [ ] 제안에 이유가 붙는가 / 거울이 판단 않고 비추기만 하는가

### 한 줄 요약

한 레이어의 시맨틱 토큰이 라이트/다크 + A/B 온도를 동시에 푼다. 모든 화면은 4상태·고르는 컴포넌트·하단 액션·바텀시트·은은한 모션을 갖추고, 매력은 체류가 아니라 *조용히 처리하고 떠나게* 하는 데서 온다.
